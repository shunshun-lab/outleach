# CLAUDE.md — Outreach App

## このアプリの目的

「発信したいもの」と「刺さる人」をつなぐ営業自動化ツール。
イベント・告知・招待を、受け手の文脈に合わせてパーソナライズし、
適切なチャネルで届ける。

## 設計原則

- ターゲット条件は AND/OR で組み合わせ可能にする
- 文章生成は**受け手の文脈を主語**にする（発信者目線にしない）
- 送信前に必ず人間の承認ステップを挟む（自動送信禁止）
- 個人情報は最小限しか保持しない（privacy.md 参照）
- コンバージョン結果は必ず progress.md に記録する

## 技術スタック

- Next.js 15 (App Router)
- TypeScript 5.x
- PostgreSQL + Prisma
- Claude API（文章生成・個人化）
- BullMQ（送信キュー管理）
- Playwright（外部プラットフォームスクレイピング）

## ディレクトリ規約

- 型定義: `src/types/`
- フィルタロジック: `src/lib/filters/`
- AI生成ロジック: `src/lib/ai/`
- 配信ロジック: `src/lib/channels/`
- APIルート: `src/app/api/`
- UIコンポーネント: `src/components/`

## Gitホスティング方針（重要）

**GitHub等の外部プラットフォームは使用しない。**

理由：
- AIによる大量コミット・リポジトリ作成がボット判定され凍結リスクがある
- コードがAIトレーニングに利用される可能性がある
- プラットフォーム側の判断で予告なくアクセスを失うリスクがある

**使用するのは自己ホストのGitea。**

- リモートURLは必ず Gitea のエンドポイントを使う
- 短時間での大量 push/repo 作成は避ける（バースト禁止）
- GitHub/GitLab へのプッシュは人間が明示的に許可した場合のみ

## セッション開始時の必須手順

1. `progress.md` を読んで現在の状態・意思決定を把握する
2. `contexts/outreach.md` を読んでドメイン知識を確認する
3. 作業完了後は必ず `progress.md` を更新する

## 禁止事項

- 承認なしの送信実行
- `privacy.md` のルールに違反する個人情報の保持
- ターゲット条件なしのブロードキャスト
- console.log に個人情報を出力すること
- GitHub/GitLab への無断プッシュ
- 短時間での大量リポジトリ作成
