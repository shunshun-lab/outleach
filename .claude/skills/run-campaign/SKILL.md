# SKILL: run-campaign — キャンペーン実行手順

キャンペーンを実行するときは、必ずこの手順に従う。
**人間の承認なしに送信を実行してはならない。**

## 手順

1. **ターゲットリストを生成する**
   - `GET /api/campaigns/[id]/preview` でマッチしたContactリストを確認する
   - 件数・内訳を人間に提示する

2. **メッセージ案を生成する**
   - `POST /api/ai/generate-message` で各Contactに個人化したメッセージを生成する
   - 最低2案生成し、スコアの高い方を選定する
   - `messaging.md` の禁止ワードが含まれていないか確認する

3. **プレビューを提示して承認を求める**
   - 宛先・件名・本文のプレビューを提示する
   - **ここで必ず人間の「OK」を待つ。自動で次に進まない。**

4. **送信キューに積む**
   - 承認後、`POST /api/campaigns/[id]/send` でキューに積む
   - BullMQが順次送信する（バースト送信禁止）
   - 送信間隔は最低5秒以上

5. **結果を記録する**
   - 送信完了後、`progress.md` のキャンペーン結果ログに記録する
   - 開封率・返信率・コンバージョン率を記録する

## 禁止

- 承認ステップをスキップする
- 全員に同じ文面を送る（個人化なし）
- 短時間での大量送信（バースト）
